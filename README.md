# Job-Inator

A client-side ATS resume optimizer. Paste a job description, and it tailors your LaTeX resume
to match it using an AI provider of your choice — no backend, everything runs in the browser.

## Features

- **Optimizer** — paste a job description, get a tailored LaTeX resume with a live PDF preview
  and download, generated by the AI provider/model of your choice.
- **LaTeX Editor** — a separate screen for editing your base resume source, with its own live
  PDF preview.
- **Multiple AI providers** — Claude (Anthropic), ChatGPT (OpenAI), Gemini (Google), and Groq.
  Each remembers its own model and API key, so switching providers doesn't lose anything.
- **Settings** — light/dark theme, and a "Save my data in this browser" toggle that controls
  whether API keys, resume source, job description, and preferences persist in localStorage or
  stay in-memory only for the current tab.

## How it works

There is no backend — whichever AI provider you pick is called directly from your browser using
your own API key for that provider. Keys are only ever sent straight to the provider you're using
(and optionally cached in this browser's localStorage, controlled by the toggle in Settings).

LaTeX rendering uses [`latex.js`](https://latex.js.org/) (a real LaTeX-to-HTML5 translator) plus
`html2canvas`/`jsPDF` for the PDF download, entirely in-browser. This supports a solid, standard
subset of LaTeX (sections, bold/italic, lists, center blocks) rather than arbitrary packages —
see `src/lib/latexCompiler.ts` and the tailoring prompt in `src/lib/providers/shared.ts` for the
exact constraints.

See [`docs/README.md`](docs/README.md) for the full documentation index — architecture, plus a
complete reference for every module under `src/`.

## Development

```bash
npm install
npm run dev      # start the dev server
npm run build    # type-check + production build
npm run preview  # preview the production build locally
npm run lint     # oxlint
```

## Security note

Because this is a pure front-end app, any API key you enter is stored in this browser's
localStorage (unless you turn off "Save my data" in Settings) and is sent directly from your
browser to the selected provider. Don't use a key with a large budget attached, and don't use
this on a shared or public computer.
